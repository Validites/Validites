# Hi there, I'm Validites! 👋

I am a passionate software developer and Android enthusiast who loves **Android Open Source Project (AOSP)**, I enjoy building several custom ROM's for all sorts of devices!

---

## 🛠️ My Tech Stack (AOSP Ecosystem)

AOSP is quite complex, these are what languages I use to help me navigate it:

### 📦 High-Level & Framework
* **Java / Kotlin:** Used heavily when interacting with the AOSP framework layers (like `frameworks/base`), system services, and building system apps.
* **Go:** Primarily utilized for configuring the modern Android build system via **Blueprint** and **Soong** (`Android.bp` files).

### ⚙️ Low-Level, Native & Kernel
* **C / C++:** My go-to for working within native daemons, hardware abstraction layers (HALs), and runtime environments (ART/Bionic). 
* **x86 / ARM Assembly:** I use assembly for low-level debugging, reverse engineering components, optimizing performance-critical code, and understanding bare-metal execution during the early boot stages.

---

## 📱 AOSP & Device Bringing Up

I don't just write code, I love seeing it run on physical hardware!

### What I do in the AOSP space:
* **Custom Builds:** For all sorts of devices (from low end Samsung's to top of the line flagships!)
* **Device Trees & Kernels:** Patching kernels, configuring device-specific trees (`device/[vendor]/[codename]`), and fixing broken hardware bindings (Wi-Fi, Bluetooth, Audio).
* **Target Architectures:** Working across both ARM64 physical targets and x86_64 emulator environments.

> 💡 **Current Project:** I am currently working on stable AOSP builds for the Xperia 5 IV (my model is the XQ-CQ54) and Samsung Galaxy A33 (my model is the SM-A336B).
